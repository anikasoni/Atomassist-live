# AtomAssist Live - Demo Script

## Goal

Show a real-time video support platform that is self-hosted, secure, and production-aware.

## 1. Open Home Page

Open:

```text
http://localhost:5173
```

Say:

```text
This is AtomAssist Live, a self-hosted real-time video support platform. It uses our own Node.js backend and a self-hosted mediasoup SFU for media routing.
```

## 2. Agent Login

Open:

```text
http://localhost:5173/agent/login
```

Login:

```text
agent@demo.com / demo123
```

Show:

- Agent dashboard
- Existing sessions
- Create Support Session button

## 3. Create Support Session

Click:

```text
Create Support Session
```

Show:

- Invite link generated
- Link copied

Say:

```text
The invite token is generated by the backend and stored as a secure hash, so the raw invite token is not stored directly.
```

## 4. Customer Join

Open invite link in another browser/profile.

Customer enters:

```text
Test Customer
```

Show:

- Customer joins call room
- Agent and customer both connected

## 5. Self-hosted SFU Video

Show:

- Local camera
- Remote participant video
- Media connected through self-hosted SFU

Say:

```text
This is not peer-to-peer. Browser media is sent to our self-hosted mediasoup SFU and routed to the other participant.
```

## 6. Chat

Send message from agent.

Send message from customer.

Show messages appear on both sides.

Say:

```text
Chat is real-time through Socket.IO and persisted to session history.
```

## 7. File Sharing

Upload a small file from agent or customer.

Show:

- File appears in chat
- Other side can download it
- File later appears in history

## 8. Reconnect Flow

Close customer tab.

Show agent sees customer status:

```text
RECONNECTING
```

Reopen customer invite/call within 60 seconds.

Show customer returns and video recovers.

Say:

```text
Disconnect is different from session end. We hold a customer reconnect window for 60 seconds.
```

## 9. End Session

Agent clicks:

```text
End Session
```

Show:

- Session becomes ENDED
- Invite link expires
- Customer cannot rejoin

Say:

```text
Only ending the session permanently expires the invite.
```

## 10. Tab Recording MVP

Agent clicks:

```text
Start Tab Recording
```

Select the AtomAssist tab/window.

Stop after a few seconds.

Show:

- Recording saved to session history
- Recording downloadable

Say:

```text
This is a browser tab recording MVP. Production SFU-side recording would consume RTP server-side and mux with FFmpeg or GStreamer.
```

## 11. Session History

Open history.

Show:

- Event timeline
- Participants
- Chat messages
- Files
- Tab recordings

## 12. Admin Dashboard

Login as admin:

```text
admin@demo.com / demo123
```

Open:

```text
http://localhost:5173/admin
```

Show:

- Total sessions
- Waiting / active / ended
- Participant count
- Session table
- History links
- Admin force-end

## 13. Observability Dashboard

Open:

```text
http://localhost:5173/admin/observability
```

Show:

- Active sockets
- HTTP requests
- Chat messages
- Files uploaded
- Tab recordings
- Memory
- Event loop lag
- Session status bars
- Raw Prometheus preview

Say:

```text
The human-friendly page is for admins, while /metrics is the raw Prometheus-compatible endpoint.
```

## Closing Line

```text
AtomAssist demonstrates a complete self-hosted WebRTC support platform with secure sessions, SFU media routing, reconnect handling, file sharing, recording MVP, admin monitoring, and production observability.
```
