# Hackathon Requirements Summary

Must-haves:
1. Agent can create call session and invite customer using link or token.
2. Both parties join from web browser.
3. System tracks who is in session.
4. Session can be ended cleanly.
5. Session history is persisted and queryable.
6. Both participants can see and hear each other in real time.
7. Media must route through a server.
8. Direct peer-to-peer is not acceptable.
9. Participants can mute audio or turn off video.
10. In-call chat works in real time.
11. Chat messages are persisted.
12. Two roles: Agent and Customer.
13. Customer joins by invite only and cannot perform agent actions.

Good-to-have:
1. Call recording.
2. File sharing.
3. Reconnect handling.
4. Admin dashboard.
5. Observability metrics.

Judging criteria:
1. Functionality.
2. Reliability.
3. Architecture.
4. User experience.
5. Good-to-have features.
6. Code quality.

Constraints:
1. Browser-based only.
2. No Twilio Video, Agora, Daily, Vonage, or hosted video APIs.
3. Media must route through your own server.
4. Must provide working demo or screen recording.
5. Must submit code repository.
6. Must provide architecture diagram.
7. Must provide README with setup and limitations.
